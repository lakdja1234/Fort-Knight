 Godot 프로젝트에 새로운 "파트"를 추가하는 방법 가이드 (파티클 제외)

  이 가이드는 기존 프로젝트의 관례를 따라 새로운 유형의 "파트"를 생성하고 게임에 통합하는 방법을 설명합니다. 여기서
  "파트"는 유도 미사일처럼 플레이어가 장착하여 사용할 수 있는 모듈형 기능을 의미합니다.

  1. "파트"의 개념 이해

  이 프로젝트에서 "파트"는 크게 두 가지 요소로 구성됩니다:

   * 파트 리소스 (`.tres` 파일): 파트의 메타데이터(이름, 설명, UI 아이콘)와 실제 게임 내 기능을 구현하는 스킬 씬에 대한
     참조를 담고 있는 데이터 파일입니다. 예를 들어, homing_missile_part.tres 파일이 이에 해당합니다.
   * 스킬 씬 (`.tscn` 파일) 및 스크립트 (`.gd` 파일): 이 씬은 파트가 게임 내에서 활성화될 때 인스턴스화되어 동작하는
     실제 게임 오브젝트입니다. 씬의 루트 노드에 연결된 스크립트가 파트의 모든 로직(예: 발사, 이동, 효과, 충돌 처리 등)을
     구현합니다. 예를 들어, HomingMissilePart.tscn과 HomingMissilePart.gd가 유도 미사일 파트의 스킬 씬과 스크립트입니다.

  이 두 가지 요소를 이해하고 기존 파일들의 구조를 따르는 것이 중요합니다. 이제 각 요소를 만드는 방법을 단계별로
  설명하겠습니다.

 2. 파트 리소스 (`.tres` 파일) 생성 가이드

  새로운 "파트"를 정의하기 위한 .tres 파일을 만드는 방법입니다. 이 파일은 게임 에디터에서 해당 파트의 속성을 쉽게 설정할
  수 있도록 해줍니다.

   1. 새 리소스 파일 생성:
       * Godot 에디터에서 res://parts/ 폴더를 마우스 오른쪽 버튼으로 클릭합니다.
       * New Resource... (새 리소스...)를 선택합니다.
       * Resource를 검색하고 선택한 후 Create (생성)를 클릭합니다.
       * 파일 이름을 [새로운파트이름]Part.tres (예: fire_blast_part.tres)로 지정하고 Save (저장)를 클릭합니다.

   2. `Part` 스크립트 연결:
       * 새로 생성된 .tres 파일을 선택합니다.
       * 인스펙터 패널에서 Resource 옆의 드롭다운 메뉴를 클릭하고 Change Type (유형 변경)을 선택합니다.
       * Part를 검색하여 선택합니다. (만약 Part가 보이지 않는다면, Script 속성에 res://parts/part.gd 파일을 드래그하여
         할당합니다.)

   3. 파트 속성 설정:
       * 스크립트가 연결되면 인스펙터에 다음과 같은 속성들이 나타납니다.
           * part_name (문자열): 이 파트의 이름을 입력합니다 (예: "화염 폭발").
           * description (문자열): 파트에 대한 설명을 입력합니다 (예: "전방에 강력한 화염 폭발을 일으킵니다.").
           * part_texture (Texture2D): 이 파트를 나타낼 텍스처 (주로 UI 아이콘)를 할당합니다. res://파츠/ 또는 적절한
             에셋 폴더에서 이미지를 드래그 앤 드롭하거나 로드합니다.
           * skill_scene (PackedScene): 이 파트가 게임 내에서 활성화될 때 인스턴스화될 .tscn 씬 파일을 할당합니다. 이
             씬은 다음 단계에서 생성할 것이므로, 현재는 비워두거나 임시로 다른 씬을 지정해도 됩니다. 나중에 올바른
             씬으로 업데이트해야 합니다.

  이 과정을 통해 새로운 파트의 메타데이터 및 스킬 씬 연결 정의가 완료됩니다.

3. 스킬 씬 (`.tscn`) 및 해당 스크립트 (`.gd`) 생성 가이드

  이 단계에서는 파트의 실제 기능을 구현할 게임 오브젝트를 만듭니다.

   1. 새 씬 생성:
       * Godot 에디터에서 Scene (씬) -> New Scene (새 씬)을 선택합니다.
       * 2D Scene (2D 씬)을 선택하여 루트 노드로 Node2D를 생성합니다.
       * 루트 노드의 이름을 파트의 기능에 맞게 변경합니다 (예: FireBlastSkill).
       * 만약 유도 미사일처럼 물리적인 움직임이나 충돌이 필요한 객체라면, 루트 노드를 RigidBody2D (발사체),
         CharacterBody2D (플레이어/적), Area2D (트리거) 등으로 변경할 수 있습니다. 기존 유도
         미사일(player_homing_missile.tscn)은 RigidBody2D를 사용했습니다.

   2. 씬 저장:
       * Scene (씬) -> Save Scene As... (씬 다른 이름으로 저장...)를 선택합니다.
       * 파일 경로를 res://skills/ 또는 res://스테이지2/ 등 적절한 위치로 지정합니다. 기존 프로젝트의 유도 미사일 씬이
         res://스테이지2/에 저장되어 있으므로, 이 경로를 따르는 것이 좋습니다.
       * 파일 이름을 [새로운스킬이름].tscn (예: fire_blast_skill.tscn)으로 지정하고 저장합니다.

   3. 스크립트 연결 및 구조화:
       * 생성된 씬의 루트 노드를 선택하고, 인스펙터 패널에서 Script 속성 옆의 아이콘을 클릭하여 New Script (새
         스크립트)를 선택합니다.
       * 스크립트 이름을 씬 이름에 맞게 [새로운스킬이름].gd (예: fire_blast_skill.gd)로 지정하고, 경로를 씬 파일과
         동일한 디렉토리에 저장합니다.
       * 스크립트 내부 구조 (예시: `HomingMissilePart.gd` 참고):
           * 상속: 루트 노드의 유형에 따라 스크립트가 상속받는 클래스를 정의합니다. 예를 들어, extends RigidBody2D.
           * 신호 (Signals): 필요하다면 cooldown_started, cooldown_finished와 같은 신호를 정의하여 다른 노드와 통신할 수
             있습니다.
           * 내보내기 변수 (`@export var`): Godot 에디터의 인스펙터에서 쉽게 조절할 수 있도록 스킬의 속성(예: damage,
             speed, cooldown)을 export 변수로 만듭니다.

   1             @export var damage: int = 15
   2             @export var speed: float = 600.0
           * `_ready()` 함수: 노드가 씬 트리에 추가될 때 한 번 호출됩니다. 초기화 작업(예: 그룹 추가, 시그널 연결, 자식
             노드 참조)을 수행합니다.
   1             func _ready():
   2                 add_to_group("skill_projectiles") # 스킬 발사체 그룹에 추가
   3                 body_entered.connect(_on_body_entered) # 충돌 처리 시그널 연결
           * `_physics_process(delta)` 또는 `_process(delta)` 함수: 매 프레임 또는 물리 프레임마다 호출됩니다. 스킬의
             움직임, 추적, 상태 업데이트 등 지속적인 로직을 구현합니다.
           * `activate()` 함수: 이 스킬 파트를 활성화할 때 호출되는 함수입니다. HomingMissilePart.gd의 activate() 함수를
             참고하여 쿨타임 체크, 발사 로직 등을 구현할 수 있습니다.

   1             func activate():
   2                 # 스킬 활성화 로직
   3                 print("New skill activated!")
   4                 # 예를 들어, 플레이어에게 발사체 생성을 요청할 수 있습니다.
   5                 # player.set_next_projectile(...)
           * 콜백 함수: 충돌(_on_body_entered), 타이머 타임아웃(_on_cooldown_timer_timeout) 등 특정 이벤트에 반응하는
             함수를 구현합니다.

  이 스킬 씬은 이제 새로운 파트의 동작을 정의하는 뼈대가 됩니다. 다음 단계에서는 이 씬에 시각적 요소와 효과를 추가하는
  방법을 설명하겠습니다.

4. 시각/효과 통합 방법 (파티클 제외)

  스킬 씬의 기본적인 로직 뼈대가 갖춰졌다면, 이제 시각적 요소를 추가하여 스킬을 시각적으로 표현하고 효과를 부여합니다.
  이는 스프라이트, 애니메이션, 충돌 영역, 쉐이더 등을 포함합니다.

   1. 스킬 시각화 (Sprite, AnimatedSprite 등):
       * 스킬 씬의 루트 노드를 선택하고 Add Child Node (자식 노드 추가)를 통해 Sprite2D 또는 AnimatedSprite2D 노드를
         추가합니다.
       * Sprite2D 노드를 사용한다면, Texture 속성에 스킬의 모습을 나타낼 이미지를 할당합니다.
       * AnimatedSprite2D 노드를 사용한다면, SpriteFrames 리소스를 새로 생성하거나 기존 것을 할당하고, 원하는 애니메이션
         클립을 추가하여 스킬의 움직임이나 변화를 표현합니다. (예: 기존 폭발 씬은 AnimatedSprite2D를 사용하여 폭발
         애니메이션을 구현했습니다.)

   2. 충돌 영역 및 물리 설정 (CollisionShape2D, Area2D 등):
       * 스킬이 다른 오브젝트와 상호작용(충돌 감지, 데미지 적용)해야 한다면 CollisionShape2D 노드를 추가합니다.
       * CollisionShape2D는 CollisionObject2D의 자식 노드여야 합니다. 루트 노드가 RigidBody2D나 Area2D인 경우 직접
         자식으로 추가할 수 있습니다.
       * Shape 속성에서 RectangleShape2D, CircleShape2D 등 적절한 모양을 선택하고 크기를 조절하여 스킬의 충돌 영역을
         정의합니다.

   3. 쉐이더 사용 (ShaderMaterial):
       * 유도 미사일의 경우 Sprite2D 노드에 ShaderMaterial을 적용하여 윤곽선 효과를 주었습니다. 이처럼 특정 노드의
         Material 속성에 ShaderMaterial을 할당하고 커스텀 Shader 리소스를 만들어 시각적 효과를 더할 수 있습니다.
       * ShaderMaterial을 적용할 노드를 선택한 후, 인스펙터의 Material 속성에서 New ShaderMaterial을 선택하고 Shader
         속성에 New Shader를 생성한 후 GDShader 코드를 작성합니다.

  이러한 시각적 요소들을 스킬 씬에 적절히 조합하여 새로운 파트의 개성을 부여할 수 있습니다.
